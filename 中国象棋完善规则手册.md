# 中国象棋完善规则手册

> 本手册综合《中国象棋基本规则》《中国象棋开发需求文档》以及《天天象棋》/中国象协标准竞赛规则，经二次检查后整理而成，用于指导网页版中国象棋游戏开发。

---

## 一、棋盘与坐标系统

### 1.1 棋盘结构
- **尺寸**：9 条竖线 × 10 条横线，共 90 个交叉点。
- **河界**：第 4 条横线（索引 4）与第 5 条横线（索引 5）之间的空白地带。
- **九宫**：两端 3×3 的"米"字格区域。
  - 红方九宫：列 3–5，行 0–2
  - 黑方九宫：列 3–5，行 7–9

### 1.2 坐标表示
- 采用 **(行, 列)** 的二维索引，行从下到上为 0–9（row 0=红方底线，row 9=黑方底线），列从左到右为 0–8。
- FEN 记录顺序：从第 9 行（黑方底线）到第 0 行（红方底线）。

### 1.3 棋子符号（WXF 标准）

| 棋子 | 红方 | 黑方 |
|------|------|------|
| 帅/将 | K | k |
| 仕/士 | A | a |
| 相/象 | E | e |
| 車/车 | R | r |
| 馬/马 | H | h |
| 炮 | C | c |
| 兵/卒 | P | p |

## 二、棋子走法规则

### 2.1 帅（将）
- 活动范围：九宫内。
- 移动方式：横或竖一格（`|dr| + |dc| = 1` 且 `dr × dc = 0`）。
- **将帅照面**：走子后，若红帅与黑将处于同一列且中间无任何棋子，则该走法**非法**。
- **主动送将**：走子后，若己方将帅可被对方任何棋子按合法几何路径攻击到，则该走法**非法**。

### 2.2 仕（士）
- 活动范围：九宫内。
- 移动方式：斜线一格（`|dr| = 1` 且 `|dc| = 1`）。

### 2.3 相（象）
- 移动方式：田字格（`|dr| = 2` 且 `|dc| = 2`）。
- 活动范围：不能过河。
  - 红相目标行必须 `≤ 4`（红方半场为 row 0-4）
  - 黑象目标行必须 `≥ 5`（黑方半场为 row 5-9）
- **塞象眼**：田字中心点（`row + dr/2`, `col + dc/2`）若有棋子，则无法移动。

### 2.4 马
- 移动方式：日字格（`|dr| + |dc| = 3`，且 `|dr| ≠ 0`，`|dc| ≠ 0`）。
- **蹩马腿**：
  - 若 `|dr| = 2`，马腿位于 (`row + dr/2`, `col`)。
  - 若 `|dc| = 2`，马腿位于 (`row`, `col + dc/2`)。
  - 马腿位置有棋子时，该方向禁止移动。

### 2.5 车
- 移动方式：横线或竖线直线移动。
- 限制：移动路径上（不含起点和终点）不得有任何棋子。
- 吃子：目标点有敌方棋子时，路径上同样不得有任何棋子。

### 2.6 炮
- 移动方式：横线或竖线直线移动。
- **不吃子**：目标点为空，且路径上棋子数为 0。
- **吃子**：目标点有敌方棋子，且路径上**有且仅有 1 个棋子**作为"炮架"。

### 2.7 兵（卒）
- **红兵**：
  - 未过河（当前行 `≤ 4`，即红方半场）：只能向前走一格（`dr = +1`, `dc = 0`）。
  - 过河后（当前行 `≥ 5`，进入黑方半场）：可向前（`dr = +1, dc = 0`）、向左（`dc = -1, dr = 0`）、向右（`dc = +1, dr = 0`）。
  - **永远不可后退**（`dr < 0` 非法）。
- **黑卒**：
  - 未过河（当前行 `≥ 5`，即黑方半场）：只能向前走一格（`dr = -1`, `dc = 0`）。
  - 过河后（当前行 `≤ 4`，进入红方半场）：可向前（`dr = -1, dc = 0`）、向左（`dc = -1, dr = 0`）、向右（`dc = +1, dr = 0`）。
  - **永远不可后退**（`dr > 0` 非法）。

## 三、胜负与和棋判定

### 3.1 将军
- 当一方走子后，其某个棋子按合法几何路径可以攻击到对方将帅时，称为"将军"。
- 被将军方必须在下一着**应将**：走出一步使己方不再被将军的着法。
- 任何导致己方被将军的走法均为**非法走法**。

### 3.2 将死（Checkmate）
- 一方被将军，且其所有棋子的**所有合法走法**均无法解除将军状态，则判该方负，对方胜。

### 3.3 困毙（Stalemate）
- 一方**未被将军**，但其所有棋子均**无任何合法走法**，则判该方负（困毙），对方胜。

### 3.4 和棋（基础版暂不强制实现，但需预留接口）
- **自然限着**：双方连续 60 回合（120 步）无吃子、无进兵，自动判和。
- **长将/长捉**：一方连续循环将军或捉子，超过规定回合数后必须变着，不变则判负；双方互不变着可判和。
- **重复局面**：同一局面出现 3 次，可判和。
- **双方同意和棋**：一方提议，对方接受。

## 四、FEN 记谱规范

每次成功落子后，系统必须生成以下格式的 FEN 字符串：

```
<棋子布局> < Active Color > < - > < - > <半回合时钟> <全回合数>
```

- **棋子布局**：从第 9 行到第 0 行，连续空位用数字 1–9 表示，行间用 `/` 分隔。
- **Active Color**：`w` 表示红方走，`b` 表示黑方走。
- **第三、四字段**：中国象棋无王车易位和吃过路兵，固定为 `- -`。
- **半回合时钟**：自上次吃子或进兵后的半回合数，用于自然限着判和。
- **全回合数**：从 1 开始，每完成黑方一着后加 1。

**初始局面 FEN**：
```
rheakaehr/9/1c5c1/p1p1p1p1p/9/9/P1P1P1P1P/1C5C1/9/RHEAKAEHR w - - 0 1
```

## 五、开发实现要点（规则引擎侧）

1. **isLegalMove(board, from, to, side)** 必须同时满足：
   - 几何路径合法（按上述各子规则）
   - 目标点为空或敌方棋子
   - 模拟走子后，己方不被将军
   - 模拟走子后，将帅不照面（已包含在将军检测中）

2. **getAllLegalMoves(board, side)** 遍历己方所有棋子，调用 isLegalMove 生成全部合法着法列表。

3. **isCheck(board, side)** 检测 `side` 一方的将帅是否处于被对方将军的状态。

4. **isCheckmate(board, side)** = `isCheck(board, side) && getAllLegalMoves(board, side).length === 0`。

5. **isStalemate(board, side)** = `!isCheck(board, side) && getAllLegalMoves(board, side).length === 0`。
